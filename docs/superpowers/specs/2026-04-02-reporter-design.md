# Reporter Implementation Design

**Date:** 2026-04-02  
**Status:** Approved

## Context

The `@playwright-cart/reporter` package is a Playwright custom reporter meant to ship test results and artifacts to the `@playwright-cart/server`. The current state is a skeleton with lifecycle hook stubs and no upload logic. The server also only has a single `POST /api/reports` stub with no implementation.

The goal is to implement real-time test result streaming: each test's result (including attachments like screenshots and traces) is uploaded to the server as soon as the test finishes. After the full run completes, the HTML report and traces are zipped and uploaded if those reporters are enabled in the Playwright config.

---

## Server API

### New Endpoints

Replaces the existing `POST /api/reports` stub.

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/runs` | Create a new run, returns `{ runId }` |
| `POST` | `/api/runs/:runId/tests` | Upload one test result (multipart) |
| `POST` | `/api/runs/:runId/report` | Upload the final HTML report zip |
| `POST` | `/api/runs/:runId/complete` | Finalize run status (no zip, when HTML reporter is disabled) |
| `GET`  | `/api/runs` | List all runs with summary metadata |
| `GET`  | `/api/runs/:runId` | Get full run details including test results |

### Request/Response Shapes

**`POST /api/runs`** ‚Äî JSON body:
```ts
{
  project: string
  branch?: string
  commitSha?: string
  startedAt: string  // ISO 8601
}
```
Response `201`: `{ runId: string }`

**`POST /api/runs/:runId/tests`** ‚Äî multipart:
- `metadata` ‚Äî JSON string:
  ```ts
  {
    testId: string         // titlePath.join(' > ') lowercased, non-alphanum ‚Üí '-', append '-retryN' if retry > 0
    title: string
    titlePath: string[]
    location: { file: string; line: number; column: number }
    status: 'passed' | 'failed' | 'timedOut' | 'skipped' | 'interrupted'
    duration: number       // ms
    errors: Array<{ message: string; stack?: string }>
    retry: number
    annotations: Array<{ type: string; description?: string }>
    attachments: Array<{ name: string; contentType: string; filename?: string }>
  }
  ```
- `attachment_0`, `attachment_1`, ‚Ä¶ ‚Äî binary file for each attachment that has a path on disk

Response `201`: `{ testId: string }`

**`POST /api/runs/:runId/report`** ‚Äî multipart:
- `report` ‚Äî zip file of `playwright-report/`
- `completedAt` ‚Äî ISO 8601 string
- `status` ‚Äî final run status: `'passed' | 'failed' | 'interrupted' | 'timedOut'`

Response `200`: `{ reportUrl: string }`

**`POST /api/runs/:runId/complete`** ‚Äî JSON body:
```ts
{ completedAt: string; status: 'passed' | 'failed' | 'interrupted' | 'timedOut' }
```
Response `200`: `{}`

**`GET /api/runs`** ‚Äî returns array:
```ts
Array<{
  runId: string
  project: string
  branch?: string
  commitSha?: string
  startedAt: string
  completedAt?: string
  status: 'running' | 'passed' | 'failed' | 'interrupted' | 'timedOut'
  testCount: number
  passedCount: number
  failedCount: number
  reportUrl?: string
}>
```

**`GET /api/runs/:runId`** ‚Äî same as above but includes `tests` array with full test metadata.

### Filesystem Layout

```
data/
  {runId}/
    run.json            ‚Üê run metadata + final status
    tests/
      {testId}.json     ‚Üê per-test result metadata
    attachments/
      {testId}/         ‚Üê attachment files for that test
    report/             ‚Üê extracted HTML report (if uploaded)
```

`runId` format: `{project}-{timestamp}` (e.g. `my-app-1743552000000`), generated by the server.

## Reporter Implementation

### Updated Options Interface

```ts
export interface PlaywrightCartReporterOptions {
  serverUrl: string             // required: e.g. 'http://localhost:3001'
  project: string               // required: identifies this project in the dashboard
  branch?: string               // optional git context
  commitSha?: string            // optional git context
  uploadConcurrency?: number    // max parallel test uploads (default: 3)
  retries?: number              // upload retry attempts (default: 3)
  retryDelay?: number           // initial backoff ms, doubles each attempt (default: 500)
}
```

### Lifecycle Hooks

**`onBegin(config, suite)`**
- POSTs `{ project, branch, commitSha, startedAt }` to `POST /api/runs`
- Stores returned `runId` for subsequent uploads
- Inspects `config.reporter` array for an `'html'` entry ‚Üí sets `this.htmlReporterEnabled`
- `onBegin` is synchronous in Playwright's interface ‚Äî start the POST but store the promise; all subsequent uploads await `this.runIdPromise` before firing

**`onTestEnd(test, result)`**
- Awaits `this.runIdPromise` to get the `runId`
- Builds multipart: `metadata` JSON + one `attachment_N` field per attachment with a `path` on disk
- Fires upload via `uploadWithRetry(...)`, pushes the resulting promise into `this.pendingUploads[]`
- Does not block ‚Äî upload runs concurrently, gated by `uploadConcurrency`

**`onEnd(result)`** (async)
- Awaits all `this.pendingUploads` to drain (so all test uploads complete before finalizing)
- If `htmlReporterEnabled`: locates `playwright-report/` (relative to `process.cwd()`), zips it using the `archiver` npm package, POSTs the zip to `POST /api/runs/:runId/report` with `completedAt` and `status`
- If `htmlReporterEnabled` is false: POSTs `{ completedAt, status }` to `POST /api/runs/:runId/complete` to finalize the run without a zip

### Retry Logic

A standalone `uploadWithRetry` function (not a method):

```ts
async function uploadWithRetry(
  fn: () => Promise<Response>,
  retries: number,
  delay: number
): Promise<void>
```

- On HTTP error or network failure: waits `delay * 2^attempt` ms, retries up to `retries` times
- On final failure: logs `[playwright-cart] upload failed after N retries: <error>` ‚Äî never throws

### Concurrency Gate

A simple semaphore based on a counter + promise queue limits simultaneous in-flight uploads to `uploadConcurrency`. This prevents overwhelming the server when many tests finish close together.

## Error Handling Summary

| Scenario | Behaviour |
|----------|-----------|
| `POST /api/runs` fails | `runId` is undefined; all test uploads are skipped; warning logged |
| Per-test upload fails after retries | Warning logged, test run unaffected |
| Report zip upload fails after retries | Warning logged, test run unaffected |
| Attachment file missing from disk | Skip that attachment, upload rest of test metadata |
| Server returns non-2xx | Treated as failure, triggers retry |

## Verification

1. Start the server: `pnpm --filter @playwright-cart/server dev`
2. Add the reporter to a local Playwright project's `playwright.config.ts` pointing at `http://localhost:3001`
3. Run the tests ‚Äî confirm `[playwright-cart]` log lines appear and `data/` directory is populated
4. `GET /api/runs` should return the run with correct counts
5. If HTML reporter is enabled, confirm `data/{runId}/report/index.html` exists and is servable
6. Simulate server downtime during a run ‚Äî confirm the test run still completes with warnings only
